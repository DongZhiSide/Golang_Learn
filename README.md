# oops

[oops](https://github.com/samber/oops): Oops 是一个全面的 Go 错误处理库, 提供结构化的错误管理和丰富的上下文信息. 它被设计为 Go 内置错误的临时替代, 增加了强大的功能, 如栈轨迹, 源代码片段, 结构化属性以及对开发者友好的调试提示.

- 🔧 可直接替换: 无缝替代标准围棋错误处理.
- 📊 丰富上下文: 添加结构化属性和用户信息.
- 🐛 调试友好: 开箱即用的栈追踪和源代码片段.
- 🔗 错误链: 包装并组合带有额外上下文的错误.
- 🛡️ 紧急恢复: 内置紧急处理和错误转换功能.
- ✅ 断言: 用于验证的一行断言辅助工具.
- ⚡ 性能: 零依赖, 轻便且快速.
- 📝 日志器集成: 支持所有主要的 Go 日志库.
- ✂️ 关注点分离: 错误处理和日志是两个独立的工作.
- 🍳 易于集成: 无需大规模重构.
